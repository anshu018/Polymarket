### 1. Dual-Path Routing Logic
* **Fast Path (Target: $< 5$ seconds)**:
  - **Trigger Condition**: News Analyst confidence $> 0.87$ AND category is pre-validated AND `resolution_keyword_cache` hit (fresh entry $< 24$ hours old) AND the event entities match cached resolution keywords.
  - **Execution**: Skips Contract Parser, Trade Decision Agent, and LLM Coordinator. 
  - **Must Still Run**: `risk_engine.py` checks and Python coordinator aggregation.
* **Full Pipeline (Target: $17\text{--}20$ seconds, hard limit $22$ seconds)**:
  - **Trigger Condition**: All other signals.
  - **Execution**: News Analyst $\rightarrow$ Contract Parser $\rightarrow$ Trade Decision $\rightarrow$ `risk_engine.py` check $\rightarrow$ Python Coordinator $\rightarrow$ LLM Coordinator (if conflict triggers).

### 2. Prompt Engineering & Memory Insertion
* **Agent Memory Integration**: When invoking the Trade Decision Agent, you must query `agent_memory` for episodic lessons triggered by the current signal's category/keywords. Prepend these lessons as a marked "Warning Block" at the very top of the model prompt, followed by the market context.

### 3. Conflict Escalation & LLM Coordinator
* Aggregate News Analyst and Trade Decision estimates.
* **LLM Coordinator Trigger**: Fired if and only if:
  1. News Analyst and Trade Decision disagree on trade direction (e.g. YES vs NO/ABSTAIN).
  2. **AND** News Analyst confidence is $> 0.70$.
* If triggered, escalate final decision to LLM Coordinator. Otherwise, use standard Python weighted average aggregation.

### 4. Supabase Timeout & Failure Fallbacks (Rule 5)
* Implement `asyncio.wait_for` wrappers set at a strict 2-second limit for all Supabase operations.
* Apply these hardcoded fallbacks upon database read timeout:
  - `resolution_keyword_cache` timeout $\rightarrow$ fallback to full pipeline.
  - `agent_memory` timeout $\rightarrow$ proceed with trade, flag `was_memoryless = true` in execution logs.
  - `idempotency_log` timeout $\rightarrow$ FAIL CLOSED (halt trading, do not submit order, log Telegram alert).
  - `open_positions` timeout $\rightarrow$ halt new trades until read succeeds.
  - `layer_c_category_versions` timeout $\rightarrow$ use hardcoded conservative defaults.

### 5. LLM Provider Timeout & Failover (Rule 6)
* When querying the Trade Decision Agent on SiliconFlow, apply a strict 18-second timeout wrapper.
* Upon timeout, immediately cancel the task, write a log entry, and trigger failover to OpenRouter (sending the exact same prompt to the backup provider).
* If both primary and fallback fail, trip the circuit breaker, halt trading, and trigger a Telegram alert.

### 6. Order Idempotency Flow (Rule 2)
* Ensure that the execution path generates a unique UUID *before* any order API call is sent.
* Write this UUID to `idempotency_log` with status `pending` *before* hitting the Polymarket CLOB.
* Verify the log first on any retries or network errors. Fail closed if Supabase is unresponsive during this check.
